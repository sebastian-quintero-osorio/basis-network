# Glossary -- Formal Specifications

| Term | Definition |
|------|-----------|
| Action | A Boolean formula with primed and unprimed variables describing one atomic step |
| Batch | A group of enterprise transactions processed together |
| Certificate of Truth | The MC_*.log file demonstrating a PASS result |
| Counterexample | A concrete execution trace violating a specified invariant |
| DAC | Data Availability Committee |
| Enterprise | A registered organization on the Basis Network |
| Invariant | A state predicate that must hold in ALL reachable states |
| Liveness Property | A temporal formula asserting "something good eventually happens" |
| Model Instance | A finite instantiation of a TLA+ spec (MC_*.tla) |
| Proof | A Groth16/PLONK cryptographic proof of batch validity |
| Research Unit | A self-contained directory tracing a feature from Research to Verification |
| Safety Property | A state predicate asserting "something bad never happens" |
| State Root | A Merkle root hash representing complete enterprise state |
| TypeOK | The type invariant constraining all variables to their declared domains |
| Validium | L2 architecture where data stays off-chain, only proofs on-chain |
