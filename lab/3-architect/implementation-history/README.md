# Implementation History

This directory contains immutable records of verified implementation units.

## Protocol

1. The Logicist completes Phase 5 (APPROVED verdict) for a research unit.
2. The walkthrough and verified TLA+ specs are copied here as a new implementation unit.
3. The Prime Architect validates the `model_check.log` (Safety Latch).
4. Implementation proceeds following the 6-phase workflow.
5. After Phase B (adversarial testing), the `ADVERSARIAL-REPORT.md` is added.

## Unit Naming Convention

```
<component>-<feature-name>/
```

Example: `state-machine-merkle-tree/`, `batch-verifier-scaled/`, `l1-state-commitment/`
