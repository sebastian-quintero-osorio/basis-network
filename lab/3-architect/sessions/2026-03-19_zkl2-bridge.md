## Decisions

1. **IBasisRollup interface**: Defined at file scope (not inside contract) per Solidity
   constraint. The interface matches BasisRollup.sol's public functions exactly.

2. **_verifyMerkleProof as view virtual**: Changed from prototype's `internal pure` to
   `internal view virtual` for test harness override. Same pattern as BasisRollup's
   `_verifyProof`. Production implementation is effectively pure.

3. **Separate Go module**: Bridge relayer at `zkl2/bridge/` has its own `go.mod`
   (separate from `zkl2/node/`). The relayer is an independent process that watches
   both L1 and L2, justifying module separation.

4. **No ADR required**: No new technology introduced. Solidity 0.8.24 and Go match
   existing stack defined in prior ADRs.

## Known Limitations

1. **Escape hatch gap**: Deposits after last finalization are stranded during escape.
   Documented limitation in TLA+ spec. Accepted trade-off.

2. **Go tests pending**: Go is not installed on the current machine. Tests are
   written and compile-ready for execution when Go is available.
