## Decisions Made

1. **10-component composite invariant**: Combined structural (wid tracking),
   balance (accounting + solvency), and escape (sequencer link) properties
   into a single inductive invariant. This avoids circular dependencies:
   e.g., EscapeHatchLiveness needs escape_solv which needs fin_gap + balance_con.

2. **Finalization gap (G1)**: Added auxiliary invariant tracking that
   `sum_l2 + pending >= sum_lastFinalized`. This is the mathematical
   foundation for proving escape solvency at ActivateEscapeHatch.

3. **Escape-sequencer link (S1)**: Added `escape_active -> seq_alive = false`
   to handle FinalizeBatch preservation cleanly (contradiction when both
   escape and seq are active).

4. **Identity refinement**: Since BasisBridge.sol was implemented directly
   from BasisBridge.tla, the refinement is the identity mapping (bisimulation).
