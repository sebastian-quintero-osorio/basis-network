## Proof Strategy

### Balance Conservation
Algebraic proof over balance deltas. Each action's net effect on total balance is zero:
- CreateAccount: 0 - 0 = 0 (dead accounts have balance 0)
- Transfer: -amount + amount = 0
- SetStorage: no balance field changes
- SelfDestruct: -balance + balance = 0 (contract to beneficiary)

Uses `sum_list_double_fupdate` from Common.v for two-entry updates.

### SMT Proof Completeness
Inductive proof on remaining tree levels. The core insight:
at each level, the parent hash is computed from the current node and its
sibling. Since siblings come from the actual tree (via `compute_node`),
the parent equals `compute_node` at the next level by definition.

The proof uses arithmetic lemmas about tree path navigation (division,
modulo, ancestor indices) to connect `verify_walk_up` with `compute_node`.

### Isolation
Direct corollaries of proof completeness + consistency invariant.
If `accountRoot = ComputeAccountRoot` (consistency), then for every
valid leaf index, `proof_completeness` guarantees the Merkle proof
generated by `sibling_hash` verifies against the root.

### Refinement
The Go implementation state IS the TLA+ specification state (identity
mapping). When Go error checks pass, the resulting state transition
is identical to the TLA+ action. Proved by case analysis on the
implementation's conditional checks.
